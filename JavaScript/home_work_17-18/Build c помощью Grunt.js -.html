Автоматизация для самых ленивых вместе с Grunt.js

В этом скринкасте, мы узнаем что такое Build и научимся автоматизировать этот интересный процесс. Рассмотрим такие вещи как: конкатенация и минификация js и css файлов, проверка на ошибки, слежка за изменениями файлов и удаление логов. И после нашей работы в этом уроке, все это процессы будут выполняться всего лишь одной командой в консоли.

Начнем с того, что такое Build. Простыми словами, Build — это сборка вашего проекта. Обычно вы пишете код приложения модульно, в раздельных файлах и с множеством комментариев (искренне надеюсь на это, что вы дорогой посетитель loft blog, именно так и поступаете). Все это делается для того, чтобы код было легче поддерживать. Но ни для кого не секрет, что чем больше файлов и чем шире комментарии, тем больше времени сервер потратит, на загрузку таких файлов. Именно поэтому на production (боевой сервер в интернете), выливается оптимизированная версия проекта, в которой все статичные файлы конкатенированы (соединены в один) и минифицированы в одну строку (удалены все пробелы, комментарии, укорочены названия переменных). Такая сжатая версия и называется Build’ом. О процессе создания такой версии проекта, сегодня речь и пойдет.

Инструмент который поможет нам с билдом, будет Grunt.js. Но минификация и конкатенация, это далеко не все, что умеет Грант. Приступим к делу и изучим этот замечательный инструмент. Для начала надо знать, что в каждом проекте где используется Grunt.js, должно быть 2 файла: package.json и Gruntfile.js. Оба должны лежать в корневой директории проекта. Но сначала установим сам Grunt, который является модулем npm (пакетного менеджера Node.js). Вводим следующие команды в консоли:

Shell
1
npm uninstall -g grunt //Эта строка нужна если у вас уже была установлена старая версия Гранта.

Shell
1
npm install -g grunt-cli

После установки создадим папку с проектом, а в нем первый файл package.json.
package.json: используется npm для хранения метаданных для проектов опубликованных как npm модули.
в нем хранятся все зависимости на дополнительные модули гранта, а также название проекта и его версия.

Shell

Теперь мы можем приступить к самому интересному. К Gruntfile.js (кстати его можно назвать и Gruntfile.coffee — если вы пишите на CoffeScript’е) , где будет описана конфигурация плагинов, определение задач, которые будут автоматизировать наши процессы и загрузка грант плагинов.

module.exports = function (grunt) {
//описываем конфигурацию
grunt.initConfig({
    pkg: grunt.file.readJSON('package.json'), //подгружаем package.json, чтобы использовать его данные
 
    jshint: {     // описываем как будет проверять наш код - jsHint
      options: {
        curly: true,
        eqeqeq: true,
        immed: true,
        latedef: true,
        newcap: true,
        noarg: true,
        sub: true,
        undef: true,
        eqnull: true,
        browser: true,
        globals: {
          jQuery: true,
          $: true,
          console: true
        }
      },
      '<%= pkg.name %>': {  //вставляем название проекта из package.json
        src: [ 'src/js/**/*.js' ]  //какие файлы надо проверять
      }
    },
 
    concat: {  //описываем работу плагина конкатенации
     dist: {
     src: ['src/js/file1.js', 'src/js/file2.js'],  // какие файлы конкатенировать
     dest: 'dest/build.js'  // куда класть файл, который получиться после процесса конкатенации
     }
    },
 
    uglify: {  //описываем работу плагина минификации js - uglify.
     options: {
     stripBanners: true,
     banner: '/* <%= pkg.name %> - v<%= pkg.version %> - <%= grunt.template.today("yyyy-mm-dd") %> */\n' //комментарий который будет в минифицированном файле.
     },
 
     build: {
     src: 'dest/build.js',  // какой файл минифицировать
     dest: 'dest/build.min.js' // куда класть файл, который получиться после процесса минификации
     }
    },
 
    cssmin: { //описываем работу плагина минификации и конкатенации css.
     with_banner: {
     options: {
     banner: '/* My minified CSS */'  //комментарий который будет в output файле.
     },
 
     files: {
     'dest/style.min.css' : ['src/css/style1.css', 'src/css/style2.css']   // первая строка - output файл. массив из строк, какие файлы конкатенировать и минифицировать.
     }
     }
    },
 
    watch: { //описываем работу плагина слежки за файлами.
     scripts: {
     files: ['src/js/*.js'],  //следить за всеми js файлами в папке src
     tasks: ['jshint', 'concat', 'uglify', 'removelogging']  //при их изменении запускать следующие задачи
     },
     css: {
     files: ['src/css/*.css'], //следить за всеми css файлами в папке src
     tasks: ['cssmin'] //при их изменении запускать следующую задачу
     }
    },
 
 
    removelogging: { //описываем работу плагина удаления логов
     dist: {
       src: "dest/build.min.js", // файл который надо отчистить от console.log
  dest: "dest/build.clean.js" // выходной файл, который получим после очистки
     }
    }
 
 
});
 
//подгружаем необходимые плагины
grunt.loadNpmTasks('grunt-contrib-jshint');
grunt.loadNpmTasks('grunt-contrib-concat');
grunt.loadNpmTasks('grunt-contrib-uglify');
grunt.loadNpmTasks('grunt-contrib-cssmin');
grunt.loadNpmTasks('grunt-contrib-watch');
grunt.loadNpmTasks('grunt-remove-logging');
 
 
//регистрируем задачу
grunt.registerTask('default', ['jshint', 'concat', 'uglify', 'cssmin', 'removelogging', 'watch']); //задача по умолчанию, просто grunt
grunt.registerTask('test', ['']); //пока пусто, но кто знает, возможно в следующих уроках мы этим воспользуемся :)
};

После тщательного составления нашей конфигурации, опробуем на деле наш процесс автоматизации. Введем в консоли всего одно слово grunt. После чего восторженно наслаждаемся результатами проделанной работы. Ведь теперь у нас есть build версия нашего проекта, полностью готовая к выливанию на production. При любых изменениях файлов, build создается заново и нам не надо делать никаких лишних действий. Автоматизация процессов во всей красе.
